Core Principles

  • Plan First, Execute Second: Most sessions start in Plan mode (shift+tab twice). Go back and forth until the plan is solid, then auto-accept.
  • Verification is Everything: Give Claude a way to verify its work. If it has that feedback loop, it will 2-3x the quality.
  • Living Documentation: Anytime we see Claude do something incorrectly, we add it to CLAUDE.md.

Example CLAUDE.md Implementation

Here is an example of setting up a local configuration file using simple TypeScript concepts.

// .claude/config.ts
import { ClaudeConfig } from '@anthropic/claude-code';

export const config: ClaudeConfig = {
  team: "Frontend Infrastructure",
  autoAccept: true,
  maxTokens: 8192,
  hooks: {
    onPostToolUse() {
      // Run local formatter
      return runPrettier();
    }
  }
};

Subagents vs. General Models

I use a few subagents regularly. Below is a comparison of typical workflows:

Workflow Agent Used Description
Code Review @claude.reviewer Identifies logical flaws and adds constraints to markdown.
Formatting code-simplifier Simplifies and refactors messy codebase structures.
End-to-End verify-app Runs the full test suite and verifies manual UI states.

Remember, there is no one correct way to use these tools. We intentionally built it in a way that you can customize it, hack it, and integrate it exactly how you prefer!
